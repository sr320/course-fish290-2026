## Mon Apr 20 — Methods & data deadline

### “Data in hand” — operational criteria (put on board)

Students should be able to answer **yes** to:

- I have the **file(s)** or database export (not just a link to a portal I haven’t downloaded).  
- I know **n** (sample size or scope) I will use for the main analysis.  
- I know **variables** (columns) for hypothesis test or comparison.  
- I can run **at least one** analysis or summary statistic **tonight** (plot, table, mean ± SD, test).

### Troubleshooting triage (instructor cheat sheet)

| Symptom | Next step |
|---------|-----------|
| “My data need an API / won’t download” | Fallback: instructor-provided **simple public dataset** list; narrow question to match |
| “Too many missing values” | Pre-register exclusion rules; report N after cleaning |
| “I don’t know what test” | Wednesday writing studio: clarify **one** primary comparison first |

## Student handout — Methods sanity checklist

Before you call Methods “done enough to move on”:

- [ ] **Data source** identified (repository, survey, agency report, etc.) with **access date** if appropriate  
- [ ] **Inclusion rules** stated (years, geography, species, lab protocol)  
- [ ] **Analysis** named at appropriate specificity (e.g., “linear regression”, “two-sample t-test”, “ANOVA”, “descriptive statistics only”)  
- [ ] **Software** and important packages or versions if required for reproducibility  
- [ ] **Ethics**: if human/animal subjects aren’t involved, say so briefly or omit per journal norms for class
